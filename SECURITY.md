# Security Policy

## Supported Versions

Security issues are addressed in the latest version on the `main` branch. Older versions are not maintained.

| Version | Supported |
|---------|-----------|
| latest (main) | ✅ |
| older releases | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability in SpikingTSF, please **do not open a public GitHub issue**. Instead, report it privately to the maintainer:

- **Email:** [jafar.bakhshaliyev@gmail.com](mailto:jafar.bakhshaliyev@gmail.com)
- **Subject line:** `[SECURITY] SpikingTSF — <brief description>`

Please include:
- A description of the vulnerability and its potential impact
- Steps to reproduce
- Any suggested mitigation if known

You can expect an acknowledgement within a few business days. We will work with you to understand and address the issue before any public disclosure.

Note: SpikingTSF is a research benchmark library and does not handle user credentials, authentication, or network services. Most security concerns are expected to relate to dependency vulnerabilities rather than the library code itself.
